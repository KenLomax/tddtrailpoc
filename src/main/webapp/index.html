A Trail Step with Dynamically included Test Code

Example step:
  1. Prerequisites
    • Maven 3.0
    • Git to clone the examples
    • Java 8 JDK to write and compile the example code, available at Oracle JDK or OpenJDK
    • Eclipse IDE or a similar development environment
  2. Install the hybris Suite locally
    1. Download and unzip the hybris 6.2.0.1 zip into a folder near the top of your file system.
    2. Navigate in to hybris-commerce-suite-6.2.0.1 and initialize the commerce suite:
      • on a Mac/Linux run .
      • on Windows run .
  3. Set up the commerce Suite in Eclipse IDE (Notes..)
    1. Import the platform and config projects into a new Eclipse Workspace
    2. Open Eclipse's Ant View and drag the platform's build.xml file into it
    3. Update Eclipse's Ant Run time to equal that of the commerce suite
    4. Try debugging the suite in Eclipse.
  4. Install and run the TDD Trail alongside the commerce suite
    1. In the folder above hybris-commerce-suite-6.2.0.1 run .
    2. Verify that you now have the folder tddtrailpoc alongside hybris-commerce-suite-6.2.0.1.
    3. In a second command line, navigate in to the folder tddtrailpoc and run
    4. Open the TDD Trail Page @ http://localhost:8080/
  5. Work through the tests on The TddTrail making them all turn green.

Can also run with mvn -Dtest=< classname >#< testname > test
  1. Create a new extension (Notes..)
    1. Navigate to hybris-commerce-suite-6.2.0.1/hybris/bin/platform, run the command ant extgen and supply the following input
      Select yempty for the extension template
      Enter the name tddtrail for the extension.
      Enter the package prefix com.hybris.tddtrail
    2. The Ant script will now create a skeleton extension for you in YOURPATH/bin/custom/tddtrail.
  2. Add the new extension to your Suite
    1. Append the new extension to config/localextensions.xml: <extensions> ... <extension name="cuppytrail" /> </extensions>
    2. Run ant to ???
  3. Confirm the extension is recognized by the Suite.
    1. Restart the commerce suite: hybrisserver.bat or ./hybrisserver.sh
    2. Open the hybris administration console http://localhost:9001/platform/extensions in a web browser and log in as admin nimda. You will get an overview on the available extensions and, if the extension comes with a core extension module, an hmc extension module, a webmodule depicted by the webroot and a core + extension module. As you can see, cuppytrail contains a core module and a web module.
  4. Import this new extension into your EclipseIDE
